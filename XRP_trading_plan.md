### 盈亏平衡计算

> 资金总量由机器人启动时从 Kraken API 动态读取，不硬编码。

- 买入后实际持仓：`balance × (1 - 0.0016)`
- 目标 **0.5% 净利润** → XRP 需上涨 **+0.82%**
- 目标 **1.0% 净利润** → XRP 需上涨 **+1.32%**

**以当前 ~$1700 为例：**
| 目标 | 每日利润 |
|------|---------|
| 0.5% | ~$8.50 |
| 1.0% | ~$17.00 |

## 每日目标达成后的行动规则

> XRP 全年 24 小时 / 7 天不间断交易，"每日" 以 UTC 00:00 为分界点重置。

### 两级目标策略

```
每笔交易同时挂两个止盈单：

  止盈A（TP1）= entry × 1.0082  →  净利 0.5%（$4）
  止盈B（TP2）= entry × 1.0132  →  净利 1.0%（$8）

  注：以 $800 主仓计算，净利 $4 = 0.4%，$8 = 0.8%
      全仓 $1000 时净利 $5 / $10 = 0.5% / 1.0%
```

### TP1 触发（达到 0.5%）时
1. **卖出 50% 仓位**，锁定 0.5% 部分利润
2. **剩余 50% 仓位**继续持有，等待 TP2
3. 将止损单上移至 **保本价（entry + 手续费）**，消除剩余风险
4. 机器人当日**不再开新仓**，专注等待剩余仓位出场

### TP2 触发（达到 1.0%）时
1. **卖出全部剩余仓位**，锁定 1.0% 净利
2. 当日**立即停止所有交易**
3. 将利润计入资金池，次日 UTC 00:00 重置计数器

### 止损触发时
1. 全部仓位按止损价出场（亏损 -0.8%）
2. 记录亏损，检查是否触发**连续 2 笔亏损**暂停规则
3. 当日亏损累计达 **-2%** → 立即停止，等待次日

### 当日目标流程图

```
开始新的一天（UTC 00:00）
  └─→ 扫描入场信号（每 60 分钟）
        ├─→ 无信号 → 继续等待
        └─→ 有信号 → 开仓
              ├─→ 止损触发 → 记录亏损 → 继续扫描（或触发暂停）
              ├─→ TP1 触发（+0.5%）→ 卖出50%，止损移至保本，等待TP2
              │     └─→ TP2 触发（+1.0%）→ 清仓，当日停止 ✅
              │     └─→ 价格回落至保本止损 → 清仓，保住TP1利润 ✅
              └─→ TP2 直接触发（+1.0%）→ 清仓，当日停止 ✅
```

## 交易策略：24/7 波段交易

> XRP 不间断交易，机器人全天候运行，每 60 分钟扫描一次机会。

### 交易参数（config.py）

```python
SYMBOL           = "XRP/USD"
TRADE_RATIO      = 0.80    # 动态仓位：每次用总余额的 80% 交易
RESERVE_RATIO    = 0.20    # 保留 20% 作为备用仓
MAKER_FEE        = 0.0016  # 0.16%
TP1_PCT          = 0.0082  # 止盈A → 净利 0.5%（卖出50%）
TP2_PCT          = 0.0132  # 止盈B → 净利 1.0%（清仓，当日停止）
SL_PCT           = 0.0080  # 止损 -0.8%
MAX_DAILY_LOSS   = 0.02    # 单日最大亏损 2%
DAY_RESET_UTC    = 0       # 每日重置时间（UTC 00:00）
CHECK_INTERVAL   = 60      # 每 60 分钟检查一次（分钟）

# 启动时动态读取：
# balance = kraken.fetch_balance()["XRP"]["free"] * xrp_price
# trade_amount = balance * TRADE_RATIO
```

### 机器人主循环逻辑（bot.py）

```
启动时：
  → 检查 Kraken 是否有未平仓位或孤立挂单
  → 若有 → 接管管理（补挂止盈/止损）
  → 若无 → 正常进入主循环

每 60 分钟执行一次（24/7 不间断）：
  1. 检查 UTC 时间，若跨日 → 重置当日盈亏计数器
  2. 检查当日是否已达到 1% 目标 → 若是，跳过直到次日
  3. 检查当日亏损是否 ≥ 2% → 若是，停止直到次日
  4. 检查账户回撤是否 ≥ 15% → 若是，停止并 Telegram 通知
  5. 检查是否触发暴跌熔断（1小时跌幅 >5%）→ 若是，暂停 4 小时
  6. 检查当前是否有持仓 → 若有，检查 TP1/TP2/止损状态
     → 检查限价买单是否挂出超过 2 小时未成交 → 若是，取消
  7. 检查趋势：EMA20 vs EMA50 → 若熊市，空仓跳过
  8. 无持仓 + 牛市 → 拉取 4H + 1H K线，计算指标
  9. 判断入场条件是否全部满足
  10. 满足 → 挂限价买单 + 预挂 TP1、TP2、止损限价单
  11. 记录日志到 trades.csv
```

### 入场条件（analysis.py）

全部满足才入场：
- RSI(1H) < 40（超卖区间）
- 当前价格在支撑位 ±0.5% 范围内
- EMA20(4H) > EMA50(4H)（上升趋势）

### 熊市处理（EMA20 < EMA50）

当 EMA20(4H) < EMA50(4H) 时，判定为下跌趋势：
- **立即空仓等待** — 不开任何新仓位
- 机器人继续每 60 分钟扫描，但跳过入场判断
- 通过 Telegram 发送通知：`⚠️ 熊市模式：空仓等待趋势恢复`
- 当 EMA20 重新回到 EMA50 之上时，恢复正常扫描并通知：`✅ 趋势恢复，恢复交易`

### 下单逻辑（strategy.py）

```
买入成交后自动挂三个限价单：
  止盈A（TP1）：entry × 1.0082  → 触发时卖出 50% 仓位
  止盈B（TP2）：entry × 1.0132  → 触发时清仓，当日停止
  止损（SL）：  entry × 0.9920  → 触发时清仓，记录亏损

TP1 触发后额外操作：
  → 取消原止损单
  → 重新挂保本止损（entry × 1.0003，覆盖手续费）
  → 当日不再开新仓
```

## 风险控制规则（risk_manager.py）

1. **单日达到 +1% 目标** → 停止当日交易，次日 UTC 00:00 继续
2. **单日最大亏损 -2%** → 取消所有挂单，停止至次日
3. **连续 2 笔亏损** → 自动暂停 24 小时
4. **TP1 触发后** → 止损上移至保本价，消除剩余风险
5. **重大新闻期间** → 手动设置 `.env` 中 `PAUSE=true` 暂停机器人
6. **熊市（EMA20 < EMA50）** → 空仓等待，不开任何新仓位
7. **暴跌熔断** → XRP 1 小时内下跌 >5%，立即取消所有挂单，暂停 4 小时，Telegram 发送警报
8. **最大回撤保护** → 账户余额从历史峰值下跌 >15%，机器人自动停止并 Telegram 通知人工介入

### 新增 config.py 参数

```python
BEAR_MARKET_PAUSE    = True   # 熊市自动空仓
FLASH_CRASH_PCT      = 0.05   # 1小时跌幅触发熔断（5%）
FLASH_CRASH_PAUSE_H  = 4      # 熔断后暂停小时数
MAX_DRAWDOWN_PCT     = 0.15   # 从峰值回撤超过15%自动停止
ORDER_TIMEOUT_H      = 2      # 限价买单未成交超时取消（2小时）
RESERVE_RATIO        = 0.20   # 备用仓缓冲，不自动部署
```

## Telegram 通知系统

### 一次性设置步骤

1. 打开 Telegram，搜索 **@BotFather**
2. 发送 `/newbot`，按指示命名你的机器人
3. 复制取得的 **Bot Token**（格式：`123456:ABCdef...`）
4. 搜索你刚建的机器人，发送任意消息启动对话
5. 浏览器打开以下网址取得你的 **Chat ID**：
   ```
   https://api.telegram.org/bot<YOUR_TOKEN>/getUpdates
   ```
   找到 `"chat":{"id":XXXXXXX}` 中的数字
6. 将两个值填入 `.env`：
   ```
   TELEGRAM_TOKEN=123456:ABCdef...
   TELEGRAM_CHAT_ID=XXXXXXX
   ```

## 策略绩效审查与改进流程

### 审查频率

| 类型 | 频率 | 时间点 |
|------|------|--------|
| 快速检查 | 每周 | 每周日 UTC 20:00 |
| 绩效深度审查 | 每月 | 每月 1 日 UTC 00:00 |
| 策略全面重估 | 每季 | 每 3 个月第 1 日 |

> 每周/每月审查可由 Claude 定时 Agent 自动生成报告并通过 Telegram 发送。

---

### 每周快速检查（每周日）

**目的：** 及早发现异常，防止问题累积一整月。

**检查内容：**
- [ ] 本周交易次数是否合理（预期 5–14 笔）
- [ ] 本周胜率是否 ≥ 55%
- [ ] 本周净利润是否为正
- [ ] 是否有连续亏损暂停记录
- [ ] Telegram 通知是否全程正常
- [ ] `trades.csv` 无缺失记录

**判断：**
- 一切正常 → 无需操作，继续运行
- 出现异常 → 标记问题，纳入当月深度审查

---

### 每月深度审查（每月 1 日）

**目的：** 评估策略整体表现，决定是否需要调整参数。

**分析指标：**

| 指标 | 健康范围 | 需要关注 |
|------|---------|---------|
| 月胜率 | ≥ 55% | < 50% |
| 月净利润 | ≥ +10% | < +5% |
| 平均每笔利润 | ≥ +0.4% | < +0.2% |
| 止损触发率 | ≤ 40% | > 50% |
| 平均持仓时间 | 2–8 小时 | > 12 小时 |
| 手续费占收入比 | ≤ 25% | > 40% |

**深度审查步骤：**
1. 导出 `trades.csv` 当月数据
2. 分析亏损交易的共同特征（时间段、市场状态、RSI 区间）
3. 与上月对比，识别趋势变化
4. 记录问题与改进建议至 `lesson_learned.md`
5. 决定是否需要参数调整（见下方改进流程）

---

### 每季策略全面重估（每 3 个月）

**目的：** 评估整体策略是否仍适合当前市场环境，考虑重大更新。

**重估内容：**
- XRP 市场结构是否改变（波动性、流动性）
- 当前技术指标参数是否仍最优（RSI 阈值、EMA 周期）
- 是否需要增加新指标或信号
- 止盈/止损比例是否需要重新校准
- 回顾全季 Lesson Learned，提炼规律

### 改进流程（任何修改上生产前必须测试）

```
发现问题 / 改进想法
  │
  ▼
记录至 lesson_learned.md
（问题描述、根因分析、改进方案）
  │
  ▼
阶段 1：在 backtest.py 中用历史数据验证改动
  → 改动后回测指标必须优于改动前
  → 通过 → 继续
  → 不通过 → 重新设计方案
  │
  ▼
阶段 2：PAPER_TRADE=true 运行 48 小时
  → 模拟结果符合预期
  │
  ▼

  → 真实盈亏符合预期
  │
  ▼

  → 更新 lesson_learned.md 记录结果
```

> **原则：任何参数改动，哪怕只改一个数字，都必须走完以上 4 个阶段。**
